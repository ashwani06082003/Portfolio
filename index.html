Problem Statement
Client needed a full-featured e-commerce and information portal for numerology services — supporting product listings, user authentication, custom calculations, and payment flows in a single cohesive platform.
Architecture
Built on ASP.NET MVC pattern with a clean separation of Controllers, Models, and Razor Views. Data access handled via ADO.NET with stored procedures in MSSQL. Stateless authentication via ASP.NET Identity with session tokens.
Tech Stack
ASP.NET Core C# MSSQL ADO.NET Razor Pages Bootstrap 5 jQuery Git
Key Features
E-commerce product listings with SQL-backed inventory, user registration & login flows, custom numerology calculator logic, payment gateway integration, responsive mobile-first design, and admin panel for content management.
Technical Challenges & Solutions
Optimized N+1 query problems in product listings by refactoring to batched stored procedures, reducing DB round-trips by ~60%. Implemented sliding session expiry with encrypted tokens to balance UX and security requirements.

Problem Statement
Manual waste sorting is inefficient and error-prone. This system automates the classification of waste items as recyclable or non-recyclable using machine learning, reducing sorting errors and enabling smarter waste management.
Architecture
A Node.js/Express backend exposes a REST API for classification requests. A trained ML model processes image inputs and returns classification predictions. Flutter mobile frontend for real-time camera input. SQL database logs classification history and analytics.
Tech Stack
Node.js Express.js Python (ML) Flutter SQL REST API
Technical Challenges
Training a reliable classification model with limited labeled data required extensive data augmentation. Cross-platform Flutter integration with the Node backend required careful API design for mobile performance constraints.
